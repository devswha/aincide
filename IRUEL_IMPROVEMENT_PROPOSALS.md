제목: 페이지 이동 없는 Paging UX 개선
카테고리: performance
영향도: high
설명: 현재 페이지네이션은 다음/이전 버튼 클릭 시 페이지 전체를 새로고침합니다. 이를 Next.js의 'useRouter'와 'useSearchParams' 훅을 사용한 클라이언트 사이드 라우팅으로 변경합니다. 페이지가 깜빡이지 않고 목록 부분만 부드럽게 업데이트되어 사용자 경험이 크게 향상됩니다.
근거: 전체 페이지 리로드는 특히 모바일 환경에서 불필요한 데이터 소모와 지연을 유발합니다. 클라이언트 사이드 내비게이션은 Next.js의 표준적인 UX 개선 패턴이며, 'Shallow Routing'을 통해 서버 데이터를 다시 호출하지 않고도 URL을 업데이트하여 일관성을 유지할 수 있습니다.

제목: PostCard 스켈레톤 UI 도입
카테고리: ux
영향도: medium
설명: 현재 사용 중인 전체 화면 로딩 스피너를 실제 PostCard 컴포넌트의 레이아웃을 본뜬 스켈레톤 UI로 교체합니다. 데이터가 로드되는 동안 사용자는 콘텐츠의 최종 형태를 미리 예측할 수 있어 로딩이 더 빠르다고 느끼게 됩니다. 이 스켈레톤 컴포넌트는 'loading.tsx' 파일에 구현합니다.
근거: Facebook, Reddit 등 주요 콘텐츠 기반 플랫폼들은 스켈레톤 로더를 적극 활용하여 인지 성능(Perceived Performance)을 높입니다. 이는 LCP(Largest Contentful Paint) 지표 개선에도 긍정적인 영향을 줄 수 있으며, 사용자에게 안정적이고 현대적인 인상을 줍니다.

제목: 실시간 게시물 업데이트 알림 기능
카테고리: feature
영향도: medium
설명: 메인 페이지에 머무는 동안, 새로운 게시물이 등록되면 "새 게시물 보기"와 같은 버튼을 표시합니다. 이 버튼을 클릭하면 페이지 상단에 새로운 게시물을 불러옵니다. 복잡한 웹소켓 대신, 주기적으로(예: 30초마다) 새 글 수를 확인하는 간단한 API 폴링(Polling) 방식을 제안합니다.
근거: 익명 커뮤니티의 핵심은 활발한 소통입니다. 사용자가 수동으로 새로고침하지 않아도 새로운 콘텐츠를 인지할 수 있게 하면 게시물 소비율과 참여도가 자연스럽게 증가합니다. API 폴링은 서버 부하가 적고 구현이 간단하여 초기 버전 기능으로 적합합니다.
